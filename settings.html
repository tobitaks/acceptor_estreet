Auto-accept order types Saved ✓

Matched against the order's Item(s) column (e.g. BPO-Exterior, BPO Interior 5 day).

City / keyword filter Saved ✓

Comma-separated, matches any, case-insensitive, whole-word. Checked against the whole order row (address/city/state if eStreet shows it in the list). Whole-word means GA matches the state GA but not CHATTANOOGA. Applied after the type filter. Leave blank to accept all.

Exclude filter / blocklist Saved ✓

Reverse of the filter above. Comma-separated, whole-word, case-insensitive. Any order whose row matches any of these is skipped. Runs after the keyword filter. Use to block specific cities/states/zips you don't want. Leave blank to exclude nothing.

Accept chance (camouflage) Saved ✓

100%

Per-order coin toss after the filters above. At 100% every matching order is accepted (no skipping). Lower it (e.g. 70%) and each order has that chance to be accepted, the rest are skipped on purpose — a 100% acceptance rate looks like a bot. Skipped orders are gone for good (not retried). Skips show in the log as Skipped (coin).

Normal-mode poll interval Saved ✓

seconds between checks

How often to poll while idle (no recent orders). Lower = catch the first order of a bulk faster, but more requests/day. When a new order appears we switch to Fast mode (~0.5s, uncapped) to grab the rest of the bulk, then fall back here after 3 min quiet. Default 20, minimum 5.

Daily accept limit Saved ✓

Stop after this many successfully accepted orders in a day (local time). When the limit is hit, the monitored tab is closed and polling stops — the session goes idle. 0 or blank = no limit. Resets at local midnight. Only Accepted orders count (Already-Taken and Failed do not).
